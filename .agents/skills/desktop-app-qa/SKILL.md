# Desktop App QA — 测试与代码审查

本 Skill 定义桌面学习计时器的质量保证流程。每次完成功能开发、修复 bug 或重构后，**必须先执行本检查清单，再声明工作完成**。

---

## 如何使用

1. 完成功能开发后，按以下 4 个板块逐项检查
2. 每项检查标记 ✅ 通过 / ❌ 失败 / ⏭️ 不适用
3. 失败项必须修复后才能继续
4. 回归测试只需运行受影响的部分

## 5. 回归测试场景

当修改涉及以下区域时，运行对应的回归测试：

### 修改计时器逻辑

```
计时核心修改 → 运行 1.1 ~ 1.6 全部功能检查

```

```
导入导出修改     → 运行 导入后检查 1.6
```

### 修改桌面集成

```

```

## 7. 常见问题诊断

| 现象 | 可能原因 | 检查项 |
|---|---|---|
| 暂停后时间继续增长 | 暂停状态未正确设置，或 getElapsed 未处理 PAUSED | 3.3, 1.2 |
| 最小化后计时停止 | setInterval 被浏览器节流，未使用时间戳方案 | 3.3.4 |
| 休眠后时间丢失 | 未持久化状态，或恢复时未正确计算休眠时长 | 1.4, 3.5 |
| 快速点击产生多条记录 | 未做防抖，或状态机未阻止重复操作 | 3.4.1, 3.4.3 |
| 重启后统计不正确 | DailySummary 未正确重算，或 Session 数据丢失 | 1.6, 3.5 |
| 高 DPI 下文字模糊 | 未使用 `-webkit-font-smoothing`，或字号未适配缩放 | 2.4 |
| 毛玻璃在浅色壁纸上不可读 | 浅色模式下文字颜色未切换，或 blur 强度不足 | 2.3.2 |
| 紧凑模式无法拖拽 | 拖拽区域在紧凑模式下被隐藏或覆盖 | 2.2.2 |
