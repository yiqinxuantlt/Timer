## 紧凑模式改造方案

### 当前问题
1. **缺少拖动区域** — 紧凑模式不渲染 TitleBar，无 `data-tauri-drag-region`
2. **缺少窗口控制** — 无法最小化/关闭，只能 Alt+F4
3. **无法恢复普通模式** — `toggleCompactMode` 无 UI 入口
4. **窗口尺寸 160×200 过小** — Controls 三个按钮(52+44+44+24间隙=164px)溢出，垂直方向 80px Ring + 28px 时间 + 44px Controls + padding 拥挤

### 新设计：220×140 横向布局

```
┌──────────────────────────────────────┐
│  [拖动区域 - 整个顶部]  [恢复] [—] [×] │
│  ┌──────┐                            │
│  │      │  00:24:32                  │
│  │  ⭕  │  paused                    │
│  │      │  [▶] [⏸] [⏹]            │
│  └──────┘                            │
└──────────────────────────────────────┘
```

### 修改清单

#### 1. 新建 `CompactTimer.module.css` — 横向布局
- 容器：flex row，居中，gap
- 左侧：TimerRing (64×64)
- 右侧：flex column，包含时间显示 + 状态标签 + 控制按钮
- Controls 按钮缩小为 36×36 (start 40×40)
- 窗口顶部添加 mini title bar：拖动区域 + 恢复按钮 + 最小化 + 关闭

#### 2. 修改 `CompactTimer.tsx`
- 添加 mini title bar：`data-tauri-drag-region` 属性包裹整个容器顶部
- 恢复按钮调用 `toggleCompactMode`
- 最小化/关闭按钮复用 TitleBar 逻辑
- 布局改为横向：Ring(64px) | TimerDisplay + Controls
- TimerRing 新增 `size="mini"` 模式 (64×64)
- Controls 新增 `size="compact"` 模式 (按钮 36×36)

#### 3. 修改 `TimerRing.tsx` + `TimerRing.module.css`
- 新增 `size="mini"`：容器 64×64，SVG viewBox 不变但缩放
- 紧凑模式下描边宽度保持 4px

#### 4. 修改 `Controls.tsx` + `Controls.module.css`
- 新增 `size` prop，`compact` 模式下按钮尺寸：
  - Start: 40×40（普通 52×52）
  - Pause/Stop: 36×36（普通 44×44）
- 间距从 `var(--space-3)` 缩小到 `var(--space-2)`

#### 5. 修改 `TimerDisplay.tsx` + `TimerDisplay.module.css`
- 新增 `size="mini"`：字体 20px，label 10px

#### 6. 修改 `App.tsx`
- 紧凑模式窗口尺寸从 `160×200` 改为 `220×140`

### 关键实现细节
- mini title bar 集成在 CompactTimer 内部，高度 28px
- `data-tauri-drag-region` 应用到 mini title bar 整个行
- 恢复按钮使用 `Maximize2` 或 `Minimize2` 图标 (lucide-react)
- 所有 Tauri API 调用（minimize/close/window resize）保持惰性 import
