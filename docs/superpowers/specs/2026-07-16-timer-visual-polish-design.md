# 计时器显示质感与静谧呼吸动效设计

**日期：** 2026-07-16
**状态：** 已获用户确认，待实现
**范围：** 计时环与时间读数的视觉细化；不改变任何计时或数据逻辑。

## 1. 目标

让学习计时器在专注进行中呈现低干扰、可持续感知的生命力。效果应增强玻璃拟态界面的层次与时间流动感，但计时数字必须始终是视觉焦点。

## 2. 已确认的视觉方向

采用「静谧呼吸」：

- 专注进行中，进度环背后的紫蓝光晕以约 4.8 秒的低频节奏呼吸。
- 进度环仅在亮度和辉光上产生轻微起伏；时间数字不缩放、不位移。
- 空闲状态保持干净静态；暂停状态为静止的琥珀提示；完成状态只播放一次绿色收束光效。
- 普通、紧凑与迷你计时环采用相同的状态语言，迷你规格降低光晕和阴影强度。

未采用的方向：明显的脉冲波纹会增加能量感但干扰长时专注；轨道流光有更强的叙事感，但会分散对数字的注意力。

## 3. 组件与样式边界

| 位置 | 职责 | 改动范围 |
| --- | --- | --- |
| `src/components/TimerRing.tsx` | 根据既有 `status` 公开稳定的视觉状态钩子 | 为容器和 SVG 补充状态样式类；不改变时间、进度或状态计算 |
| `src/components/TimerRing.module.css` | 计时环、光晕、状态动效与时间读数细节 | 新增低频呼吸、暂停、完成和规格缩放样式 |
| `src/components/CompactTimer.module.css` | 紧凑视图的数字读数一致性 | 仅补充小尺寸读数的精细阴影或过渡，保持布局不变 |
| `src/styles/globals.css` | 应用容器的状态氛围 | 微调运行和完成状态的背景/边框过渡；复用既有设计令牌 |

不引入动画库、新组件、状态字段、计时器或持久化变更。

## 4. 视觉状态规范

| 状态 | 计时环 | 光晕与容器 | 数字/文案 |
| --- | --- | --- | --- |
| `IDLE` / `CANCELLED` | 静态渐变进度与基准阴影 | 无循环动效 | 读数清晰、状态文字使用静音色 |
| `RUNNING` | 原有进度过渡不变，辉光轻微呼吸 | 4.8 秒 `transform` + `opacity` 呼吸；应用容器保留极浅紫蓝氛围 | 时间使用等宽数字、精细字距与极浅投影；不参与动画 |
| `PAUSED` | 停止循环动画，使用稳定暖色强调 | 无缩放、无闪烁 | 状态文字为警示色，确保一眼可辨 |
| `COMPLETED` | 绿色完成渐变与单次收束 | 约 500 ms 的一次性光晕扩散/回落 | 完成颜色清晰，效果结束后回归静态 |

## 5. 动效和可访问性约束

- 循环和一次性动画仅使用装饰层的 `transform` 与 `opacity`；SVG `filter` 仅作为静态辉光，不参与动画。不得使用布局动画或 JavaScript 定时器。
- 保持现有 `stroke-dashoffset` 进度过渡；禁止以 `setInterval` 驱动任何计时显示。
- 所有循环动画仅在 `RUNNING` 状态激活，默认节奏约 4.8 秒，采用项目的 `--ease-out`。
- `prefers-reduced-motion: reduce` 下禁用循环和完成动效，仅保留状态颜色变化。
- 光晕必须放在可访问内容之后或标为装饰，不能改变读数、状态标签或按钮的语义。
- 高 DPI 与浅色主题下，透明度和辉光必须保持柔和，不能造成文字对比下降。

## 6. 数据流与错误处理

视觉层只消费已有的 `TimerRing` 属性：`progress`、`status` 与 `elapsed`。`useTimer`、`timerStore`、统计和本地存储均不接触。

若状态样式缺失或浏览器不支持某个视觉增强，组件仍展示原有静态圆环和时间文本；动画是渐进增强，而非完成计时功能的前提。

## 7. 验证标准

1. 普通视图：分别检查空闲、运行、暂停、完成状态；运行时数字稳定且呼吸不突兀。
2. 紧凑视图：检查迷你环和外部时间读数；动效不挤压现有横向布局。
3. 外观：检查深色、浅色、高 DPI 缩放与应用窗口边界，确认没有闪烁或过强发光。
4. 可访问性：开启系统“减少动态效果”后，循环动效和完成动画均不播放。
5. 工程检查：运行 `npm run typecheck`、`npm run build`、`cargo check --manifest-path src-tauri/Cargo.toml`，并按桌面应用 QA 清单验证开始、暂停、停止、持久化、窗口与交互行为。
